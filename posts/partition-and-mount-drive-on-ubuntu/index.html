Partition and Mount Drive on Ubuntu 紀錄

Jul 13, 2019
1 min read
ubuntupartitionpartedgptlinux

前提: 砌機的人將第二Hard disk(hdd)弄成Window能讀的ntfs格式, 另外有兩個partition, 其中一個是MS Reserved。

目標: 整合第二Hard disk Partitions, format為ext4, mount在/data, current user要有權限操作, 在nautilus加個bookmark

  1. df -Th

    看看目標的Drive是哪個, 有沒有掛載, 我是/dev/sdb1

    1. sh unmount /dev/sdb1 
  2. sudo -i

    取得root權限

    1. sh parted /dev/sdb

  3. (in parted) print 看看有甚麼Partitions

  4. (in parted) rm 所有Partitions

  5. (in parted) mkpart data 2048s 100% 我也不太清楚為甚麼從2048開始, gpt的table好像需要一些offset, 資料太多時間太少, 改日研究

  6. mkfs.ext4 /dev/sdb1

  7. mount /dev/sdb1 /data

    別人都mount在/mnt, 我也不太清楚直接mount在/data有甚麼不好, 如果有問題到時改回去/mnt, 然後用ln -s 創建symbolic link

    1. sh nano /etc/passwd 看看自己用的user叫甚麼, 我是aoi
  8. chown -R aoi:aoi /data

    取得folder權限

  9. 設定開機自動掛載 blkid 看看Partition的UUID /dev/sdb1: UUID=“xxxxxxxxxx” TYPE=“ext4” PARTLABEL=“data” PARTUUID=“yyyyyyyyyy”

  10. cp /etc/fstab /etc/fstab.old

    backup 開機掛載設定

    1. sh nano /etc/fstab

最底新增

# /data is on /dev/sdb1
UUID=xxxxxxxxxx       /data   ext4    errors=remount-ro       0       2

(到這裡重開機已經看得到/data)

  1. (回到desktop) 開啟nautilus, 隨便打開一個folder, 用bkspace回到/, 找到/data 然後bookmark

note: 1. lost+found的folder是ext4格式自帶的系統檔的樣子, 盡量不刪除 2. df -Th 看到的Type是fuseblk就是ntfs
